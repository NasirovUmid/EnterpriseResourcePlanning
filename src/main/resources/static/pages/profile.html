Role Routing

Each role now gets its own front-end path.

`USER` works through personal pages. `MOD_PROJECTS` and `MOD_ORGANIZATIONS` get their own workspace pages. `ADMIN` can also open the combined admin panel.

  • Token refresh stays automatic in the background.
  • UI visibility is based on `GET /users/{id}/roles`.
  • Backend security still remains the real source of truth.
