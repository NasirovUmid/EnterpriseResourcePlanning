Automatic Session

Token refresh is now background behavior.

Protected requests first try the current access token. If it is expired, the page silently calls `/auth/refresh`. If refresh also fails, the session is cleared and you are redirected to login.

  • No manual refresh button is needed anymore.
  • Reloading profile also reloads organizations and projects.
  • The profile still needs a valid `userId` because your current API is based on `GET /users/{id}`.
